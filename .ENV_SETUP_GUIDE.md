# .env.local Setup - Step by Step

## What You Need to Add

Create a file named `.env.local` in the root of your project (same level as package.json):

```
biyanis-main/
├── src/
├── package.json
├── .env.local          ← Create this file here
├── vite.config.js
└── ...
```

---

## Complete .env.local File Content

Copy and paste this into your `.env.local` file:

```env
# Supabase Configuration (REQUIRED)
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

# Gemini API Key (Optional but recommended for chatbot)
VITE_GEMINI_API_KEY=AIzaSyD...

# App Configuration (Optional)
VITE_APP_NAME=Biyanis EdTech
VITE_ADMIN_PASSWORD=Biyanis@123
```

---

## How to Get Each Value

### 1️⃣ VITE_SUPABASE_URL

**Steps:**
1. Go to your Supabase project dashboard
2. Click **Settings** (bottom left sidebar)
3. Click **API** tab
4. Copy the **Project URL**
5. Example: `https://abc123xyz.supabase.co`

**Paste in .env.local:**
```env
VITE_SUPABASE_URL=https://abc123xyz.supabase.co
```

---

### 2️⃣ VITE_SUPABASE_ANON_KEY

**Steps:**
1. Still in **Settings → API** page
2. Look for **API Keys** section
3. Find the row labeled **anon public**
4. Click the copy icon (📋) next to it
5. Example: `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9eyJyb2xlIjoiYW5vbiIsImF...`

**Paste in .env.local:**
```env
VITE_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9eyJyb2xlIjoiYW5vbiIsImF...
```

---

### 3️⃣ VITE_GEMINI_API_KEY (Optional)

**For the AI Chatbot feature. Get from Google:**

1. Go to [Google AI Studio](https://makersuite.google.com/app/apikey)
2. Sign in with Google account
3. Click **Create API Key**
4. Copy the key
5. Example: `AIzaSyD1x2y3z4a5b6c7d8e9f0g1h2i3j4k5l6m7n8o`

**Paste in .env.local:**
```env
VITE_GEMINI_API_KEY=AIzaSyD1x2y3z4a5b6c7d8e9f0g1h2i3j4k5l6m7n8o
```

⚠️ **Note:** The chatbot won't work without this key, but the rest of the app will be fine.

- ✅ Create `.env.local` in project root
- ✅ Add to `.gitignore` (already done)

4. **Check browser console**

   - Or no warning about missing credentials

## Troubleshooting

### "Supabase credentials missing" warning
**Solution:**
- Check `.env.local` exists in project root
- Verify `VITE_SUPABASE_URL` value is correct
- Verify `VITE_SUPABASE_ANON_KEY` value is correct
- Restart dev server: `npm run dev`

### Still seeing warning?
```bash
# Make sure dev server stopped
Ctrl+C

# Delete any cache
rm -rf node_modules/.vite

# Restart
npm run dev
```

### "Invalid API key" error
- Copy entire anon key from Supabase (not partial)
- Make sure no extra spaces or quotes
- Try copying again from Supabase dashboard

### Site loads but "no data"
- Verify database tables created (check Supabase SQL Editor)
- Check if RLS policies allow SELECT
- Open DevTools → Network tab → check for errors
